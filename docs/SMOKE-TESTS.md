# Smoke-тесты перед релизом



Ручной чек-лист кросс-браузерной проверки ИТ-портала. Выполняйте перед выкладкой в prod или после крупных изменений.



## Браузеры



| Платформа | Браузер | Минимальная версия |

|-----------|---------|-------------------|

| Desktop Windows | Chrome / Edge | актуальная − 1 |

| Desktop | Firefox | актуальная − 1 |

| iOS | Safari | актуальная − 1 |

| Android | Chrome | актуальная − 1 |



## 1. Поиск



- [ ] Запрос из 3+ символов фильтрует карточки

- [ ] Синонимы работают (напр. «принтер» → карточки МФУ)

- [ ] Опечатка раскладки (напр. «ghbdtn» → «принтер»)

- [ ] Пустая выдача — empty state и кнопка сброса

- [ ] Escape сбрасывает поиск (модалка закрыта)

- [ ] Ctrl/Cmd+F фокусирует поле поиска (не открывает браузерный Find)

- [ ] Клавиша `/` фокусирует поле поиска (fallback)

- [ ] Пустые section-group скрываются при фильтрации



## 2. Навигация



- [ ] Chip-навигация прокручивает к разделу

- [ ] Активный chip подсвечивается при ручном скролле

- [ ] Кнопка «наверх» появляется после прокрутки вниз

- [ ] Клик «наверх» / заголовок портала возвращает на главную



## 3. Формы заявок

- [ ] `tech_support` / `software_issues`: местоположение и детальное описание

- [ ] Валидация обязательных полей

- [ ] Сброс пароля: три поля, валидация



## 4. Модалки и a11y

- [ ] Карточки: Tab + Enter/Space открывают форму

- [ ] Skip-link «Перейти к содержимому» работает с клавиатуры



## 5. Тема



- [ ] Переключение light/dark

- [ ] `prefers-reduced-motion`: без лишних анимаций



## 6. Отправка заявок



### demoMode: true (по умолчанию)



- [ ] Submit блокируется на время отправки (нет двойного клика)

- [ ] Демо-сообщение в модалке, данные в консоли (F12)

- [ ] Модалка закрывается после успеха



### demoMode: false (prod + backend proxy)



- [ ] Успех: сообщение с номером задачи (ITHELP-NNN)

- [ ] Ошибка сети / 4xx / 5xx: текст ошибки, модалка **остаётся открытой**, данные сохранены

- [ ] Двойной клик submit не создаёт дубликат

- [ ] Password reset: отдельный flow через `trackerResetApiUrl`

## 8. Мини-экскурсия (tour)



По умолчанию `tour.enabled: true`. Для отключения: `tour: { enabled: false }` в `config.local.js`.



- [ ] `enabled: false` — overlay не появляется, кнопка «Тур» скрыта

- [ ] Первый визит (очистить `portal-tour-v1` в localStorage) → автостарт при `autoStart: true`

- [ ] «Пропустить» / Esc → тур закрывается, после reload не показывается

- [ ] «Готово» на последнем шаге → после reload не показывается

- [ ] `?tour=reset` → сброс и повторный показ

- [ ] `?tour=1` → принудительный старт без сброса storage

- [ ] Кнопка «Тур» в шапке при `showReplayButton: true` → повторный показ

- [ ] Tab не выходит за пределы popover тура

- [ ] `prefers-reduced-motion`: без анимации spotlight

- [ ] Открытая модалка заявки → тур приостанавливается, после закрытия продолжается

## 10. Авторизация Yandex 360

Запуск через `npm start` (backend + статика). Инструкция: [AUTH-SETUP.md](AUTH-SETUP.md).

- [ ] `auth.enabled: false` — портал без gate, как раньше
- [ ] `requireAuth: true` — без сессии виден только экран «Войти через Яндекс»
- [ ] Вход @21vek.by — портал, аватар в шапке, меню «Выйти»
- [ ] Форма заявки — поле ФИО заполнено (`autoFillFio: true`)
- [ ] Сброс пароля — «ФИО, кто запрашивает» заполнено
- [ ] Logout — снова gate
- [ ] `requireAuth: false` — портал открыт, опциональный вход через «Войти» в шапке
- [ ] Email не @21vek.by — сообщение об ошибке на gate
- [ ] `errors/401.html` — кнопка «Войти» ведёт на `/api/auth/login`

## 11. Безопасность API

- [ ] `POST /api/tracker/issues` без сессии возвращает `401`
- [ ] `POST /api/tracker/password-reset` без сессии возвращает `401`
- [ ] `POST /api/auth/logout` c чужого Origin отклоняется (`403`)
- [ ] Rate limit срабатывает на спаме `/api/auth/login` (получаем `429`)
- [ ] В production `GET /api/auth/config-check` не раскрывает `redirectUri` и `allowedEmailDomain`
- [ ] CSP / `X-Frame-Options` / `X-Content-Type-Options` присутствуют в ответах
