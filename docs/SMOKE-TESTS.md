# Smoke-тесты перед релизом

Ручной чек-лист кросс-браузерной проверки ИТ-портала. Выполняйте перед выкладкой в prod или после крупных изменений.

## Браузеры

| Платформа | Браузер | Минимальная версия |
|-----------|---------|-------------------|
| Desktop Windows | Chrome / Edge | актуальная − 1 |
| Desktop | Firefox | актуальная − 1 |
| iOS | Safari | актуальная − 1 |
| Android | Chrome | актуальная − 1 |

## 1. Поиск

- [ ] Запрос из 3+ символов фильтрует карточки
- [ ] Синонимы работают (напр. «принтер» → карточки МФУ)
- [ ] Опечатка раскладки (напр. «ghbdtn» → «принтер»)
- [ ] Пустая выдача — empty state и кнопка сброса
- [ ] Escape сбрасывает поиск (модалка закрыта)
- [ ] Ctrl/Cmd+F фокусирует поле поиска
- [ ] Пустые section-group скрываются при фильтрации

## 2. Навигация

- [ ] Chip-навигация прокручивает к разделу
- [ ] Активный chip подсвечивается при ручном скролле
- [ ] Кнопка «наверх» появляется после прокрутки вниз
- [ ] Клик «наверх» / заголовок портала возвращает на главную

## 3. Формы заявок

- [ ] Все типы открывают модалку с корректным заголовком
- [ ] `hr_new`: двухшаговый мастер (шаг 1 → шаг 2 → отправка)
- [ ] `tech_support` / `software_issues`: местоположение и детальное описание
- [ ] Валидация обязательных полей
- [ ] Сброс пароля: три поля, валидация

## 4. Модалки и a11y

- [ ] Escape закрывает модалку
- [ ] Клик по backdrop закрывает
- [ ] Tab не выходит за пределы модалки (focus trap)
- [ ] После закрытия фокус возвращается на карточку
- [ ] Карточки: Tab + Enter/Space открывают форму
- [ ] Skip-link «Перейти к содержимому» работает с клавиатуры

## 5. Тема

- [ ] Переключение light/dark
- [ ] Сохранение после перезагрузки
- [ ] Синхронизация между вкладками
- [ ] `prefers-reduced-motion`: без лишних анимаций

## 6. Отправка заявок

### demoMode: true (по умолчанию)

- [ ] Submit блокируется на время отправки (нет двойного клика)
- [ ] Демо-сообщение в модалке, данные в консоли (F12)
- [ ] Модалка закрывается после успеха

### demoMode: false (prod + backend proxy)

- [ ] Успех: сообщение с номером задачи (ITHELP-NNN)
- [ ] Ошибка сети / 4xx / 5xx: текст ошибки, модалка **остаётся открытой**, данные сохранены
- [ ] Двойной клик submit не создаёт дубликат
- [ ] Password reset: отдельный flow через `trackerResetApiUrl`

## 7. Конфигурация среды

- [ ] `js/config.local.js` подключён на prod/stage (см. [config.local.example.js](../js/config.local.example.js))
- [ ] URL «Полезное» (CMDB, справочник, Wiki) ведут на ресурсы среды
