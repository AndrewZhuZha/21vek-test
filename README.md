# ИТ-портал 21vek

Корпоративная стартовая страница для сотрудников: единая точка входа в ИТ-сервисы компании. Портал помогает быстро найти нужную услугу, оформить заявку в Яндекс Трекер или перейти к полезным внутренним ресурсам — без лишних переходов и поиска «куда писать».

Проект представляет собой статический сайт без серверной части и базы данных. Вся логика работает в браузере; после развёртывания на веб-сервере или открытия локально страница готова к использованию.

---

## Для кого этот портал

- **Сотрудники** — чтобы оформить обращение в ИТ, найти нужную форму или перейти к CMDB, справочнику, Wiki.
- **HR и руководители** — для типовых сценариев: новый сотрудник, увольнение, изменение данных, оргструктура.
- **Команда ИТ** — как единый витринный слой перед очередью заявок в Трекере; настраивается без переписывания всего интерфейса.

## Что можно делать на портале

### Карточки услуг и разделы

Главный экран разбит на тематические блоки. Каждый блок содержит карточки с кратким описанием услуги. Клик по карточке открывает форму заявки или внешнюю ссылку.

| Раздел | Что внутри |
|--------|------------|
| Техподдержка и сбои | Консультации, помощь с ПО, сообщения о сбоях и ошибках |
| Оборудование | Выдача, возврат и передача техники между сотрудниками |
| HR и кадры | Новый сотрудник, увольнение, изменение данных, оргструктура |
| Серверы и сеть | Создание ВМ, VPN, сетевой доступ, VLAN |
| СКУД и видеонаблюдение | Пропуска, ремонт СКУД, установка камер |
| Принтеры | Настройка и ремонт МФУ/принтеров |
| Другие вопросы | Универсальное обращение и свободный запрос |
| Полезное | Быстрые ссылки на SMDB (Snipe-IT), телефонный справочник, Wiki |

### Заявки в Яндекс Трекер

При выборе карточки открывается модальное окно с формой. Пользователь заполняет ФИО, краткое описание, подкатегорию и дополнительные поля — они зависят от типа заявки (местоположение, детальное описание, HR-атрибуты и т.д.).

Для сценария **«Новый сотрудник»** предусмотрен двухшаговый мастер: сначала базовые данные, затем выбор сервисов и доступов.

Готовая заявка формируется под очередь **ITHELP** в Яндекс Трекере. В демо-режиме отправка имитируется: данные выводятся в консоль браузера, окно закрывается автоматически. В рабочем режиме подключается реальная интеграция с API Трекера.

### Сброс пароля

Отдельная форма доступна из шапки. Запрашиваются ФИО сотрудника, инициатор запроса и причина. Работает по тому же принципу модальных окон, что и основные заявки.

### Поиск по услугам

Строка поиска в шапке фильтрует карточки на лету. Поиск учитывает:

- названия и описания карточек;
- синонимы и разговорные формулировки (например, «принтер» и «МФУ», «VPN» и «удалёнка»);
- опечатки раскладки (русская/латинская);
- ключевые слова разделов.

При вводе показывается индикатор обновления результатов; пустая выдача сопровождается подсказкой и кнопкой сброса. Словарь поиска редактируется отдельным конфигурационным файлом и пересобирается служебным скриптом — без правки основного кода портала.

### Навигация

- **Лента разделов** под шапкой — быстрый переход к нужному блоку на странице; при прокрутке подсвечивается текущий раздел (scroll-spy).
- **Клик по названию «ИТ-портал 21Vek»** — возврат на главный экран: сброс поиска и прокрутка вверх.
- Ссылки **WIKI**, **Обучение** в шапке ведут на внутренние ресурсы компании.

### Темы оформления

Поддерживаются светлая и тёмная темы. Выбор сохраняется в браузере и синхронизируется между вкладками. Переключение сопровождается плавной анимацией; учитывается системная настройка «уменьшить движение».

### Удобство работы

- **Ctrl+F** (Cmd+F на Mac) — фокус на поиск портала, а не браузерный поиск по странице.
- **Esc** — сброс активного поиска (если не открыто модальное окно).
- **Крестик, Esc, клик по затемнению** — закрытие любого модального окна.
- Кастомные полосы прокрутки и выделение текста в фирменных цветах 21vek.
- Адаптивная вёрстка: удобно на десктопе, планшете и телефоне.

## Как устроен проект (без технических деталей)

| Часть | Назначение |
|-------|------------|
| Главная страница | Разметка разделов, карточек, форм и модальных окон |
| Стили | Внешний вид, темы, анимации, адаптивность |
| Конфигурация | Список разделов, ссылки «Полезное», параметры темы, режим демо |
| Типы заявок | Описание форм и подкатегорий (`data/request-types.json`) |
| Логика заявок | Валидация, сбор данных для Трекера |
| Поиск | Индекс карточек и словарь синонимов |
| Сборка поиска | Автоматическое обновление поискового индекса после изменений |

Секреты и ключи API в репозиторий не попадают. Интеграция с Трекером предполагается на стороне бэкенда или прокси; фронтенд готовит структурированный payload заявки.

## Запуск и развёртывание

Достаточно разместить файлы проекта на любом статическом хостинге (nginx, IIS, GitHub Pages, внутренний веб-сервер) или открыть главную страницу через локальный HTTP-сервер. Сборка фронтенда не требуется.

В папке `errors/` лежат страницы типовых HTTP-ошибок: `400`, `401`, `403`, `404`, `500`, `502`, `503`, `504`. Их нужно указать в конфигурации веб-сервера (пример для nginx: `error_page 404 /errors/404.html;`). Подробнее — в `errors/DEPLOY.md`. Пересборка: `node scripts/build-error-pages.mjs`.

После изменения словаря поиска или типов заявок нужно один раз пересобрать артефакты служебным Node-скриптом из папки scripts (нужен установленный Node.js):

```bash
node scripts/build-search-index.mjs
```

Скрипт обновляет `js/search-index.js` и `js/request-types.js`.

## Настройка под вашу среду

Через конфигурационный файл можно изменить:

- названия и состав разделов навигации;
- URL полезных ссылок и внешних ресурсов (Wiki, SMDB, справочник);
- очередь Трекера и список двухшаговых форм;
- тему по умолчанию и демо-режим.

Через словарь поиска — ключевые слова, синонимы и привязку запросов к карточкам, чтобы сотрудники находили услуги привычными формулировками.

Новые типы заявок добавляются в три шага:

1. Карточка на главной странице (`data-request-type`).
2. Описание формы в `data/request-types.json`.
3. Пересборка: `node scripts/build-search-index.mjs`.

Подробнее — в [docs/IMPROVEMENTS.md](docs/IMPROVEMENTS.md) (раздел ARCH-02).
